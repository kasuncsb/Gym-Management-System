
Gym Management System - Textual Representation of Diagrams
==========================================================

1) Entity Class Diagram
-----------------------

Enums:
- SubscriptionStatus: ACTIVE, EXPIRED, CANCELLED
- AppointmentStatus: SCHEDULED, COMPLETED, CANCELLED
- EventType: IN, OUT
- PaymentStatus: SUCCESS, FAILED, PENDING, REFUNDED
- StaffRole: TRAINER, MANAGER, ADMIN

Classes / Entities:

User (base)
- Attributes:
  - userId: String
  - firstName: String
  - lastName: String
  - email: String
  - dateOfBirth: LocalDate
- Methods:
  - getFullName(): String

Member (inherits User)
- Attributes:
  - qrCodeData: String
  - joinDate: LocalDate
  - emergencyContact: String
- Methods:
  - generateNewQRCode(): void
  - hasActiveSubscription(): boolean

Staff (inherits User)
- Attributes:
  - staffId: String
  - hireDate: LocalDate
  - role: StaffRole
- Methods:
  - isCurrentlyWorking(): boolean

Trainer (inherits Staff)
- Attributes:
  - specialization: String
  - certification: String
- Methods:
  - getAvailableTimeSlots(date: LocalDate): List<LocalTime>

Manager (inherits Staff)
- Attributes:
  - department: String
  - accessLevel: int
- Methods:
  - generateStaffReport(): StaffReport
  - viewInsights(): InsightReport
  - generateDetailedReport(): Report

Admin (inherits Staff)
- Attributes:
  - systemAccessLevel: int
- Methods:
  - updateSystemConfiguration(): boolean
  - manageUserRoles(): boolean
  - createSubscriptionPlan(): void

Subscription
- Attributes:
  - subscriptionId: String
  - planName: String
  - price: double
  - status: SubscriptionStatus
- Methods:
  - isValid(): boolean
  - hasEntitlement(entitlement: String): boolean

Payment
- Attributes:
  - paymentId: String
  - amount: double
  - date: LocalDateTime
  - status: PaymentStatus
- Methods:
  - isSuccessful(): boolean

WorkoutPlan
- Attributes:
  - planId: String
  - name: String
  - description: String
- Methods:
  - markComplete(): void

Appointment
- Attributes:
  - appointmentId: String
  - date: LocalDate
  - startTime: LocalTime
  - status: AppointmentStatus
- Methods:
  - isUpcoming(): boolean
  - cancel(): void

CheckInLog
- Attributes:
  - logId: String
  - timestamp: LocalDateTime
  - eventType: EventType
- Methods:
  - isCheckIn(): boolean
  - isCheckOut(): boolean

Key Relationships:
- Inheritance:
  - Member and Staff inherit from User.
  - Trainer, Manager, Admin inherit from Staff.
- Member ↔ Subscription:
  - A Member may have multiple Subscriptions (Member 1 — * Subscription)
- Subscription ↔ Payment:
  - Payments are associated with subscriptions; Payment tracks transaction status.
- Member ↔ Payment:
  - A Member can have multiple Payment records (history).
- Member ↔ Appointment:
  - A Member can have many Appointments (Member 1 — * Appointment)
- Trainer ↔ Appointment:
  - A Trainer can have many Appointments (Trainer 1 — * Appointment)
- Member ↔ WorkoutPlan:
  - Each Member can be associated with a WorkoutPlan (primary plan or many depending on use).
- CheckInLog ↔ User/Member:
  - Many logs per user (User 1 — * CheckInLog)
- Notes:
  - Some dashed lines in the diagram indicate weaker dependencies (e.g., Payment ↔ Subscription).
  - Multiplicities inferred from domain semantics where labels were too small to read.

AccessController
- Attributes:
  - accessLog: List<AccessEvent>
  - unlockDuration: Duration
  - emergencyOverride: boolean
- Methods:
  - unlockDoor(): boolean
  - lockDoor(): boolean
  - emergencyUnlock(): boolean
  - getAccessHistory(): List<AccessEvent>
  - setAutoLockTimeout(seconds: int): void

Controller Relationships (high-level):
- SchedulingController uses NotificationController and PaymentController.
- SubscriptionController interacts with PaymentController and NotificationController.
- QRScanController interacts with AuthenticationController, TimestampController, and AccessController.
- TimestampController used by reporting and access controllers.
- NotificationController used by many controllers for alerts and reminders.

Admin / Manager portals (interfaces)
- WebAdminPortal:
  - Attributes: adminUser: Admin, systemAccessLevel: int, systemConfig: SystemConfig
  - Methods: manageSystemSettings(settings), manageUserRoles(...), createSubscriptionPlan(...), viewAuditLog(), etc.
- ManagerPortal:
  - Attributes: managedTrainers: List<Trainer>, department: String, accessLevel: int
  - Methods: viewInsights(), generateReport(reportType, params), manageTrainerSchedules(), viewStaffPerformance(), approveExpenses(), manageMemberSubscriptions()

Interface Relationships (high-level):
- MobileAppUI depends on QRScannerService, PaymentGateway, SchedulingService, NotificationService, AuthenticationService, and SubscriptionService.
- QRScannerService is used by QRScanController and MobileAppUI.
- PaymentGateway is used by PaymentController / PaymentService.
- DoorLockService is used by AccessController / TimestampController.
- AuthenticationService is central and used by most controllers for session validation.
- NotificationService is used broadly by scheduling, subscription and payment workflows.
